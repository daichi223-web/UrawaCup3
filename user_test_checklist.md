# ユーザー受入テスト (UAT) チェックリスト

このチェックリストは、エンドユーザー視点で `final_day_results.html` が「正しく動いているか」を検証するための手順書です。

## 🧪 テスト実施の準備
1.  `final_day_results.html` をブラウザ（Chrome/Edge推奨）で開く。
2.  `final_day_schedule.json`（またはサンプルデータ）を読み込み、入力画面が表示されている状態にする。

---

## 1. 基本フロー確認 (Happy Path)

「普通に使って、正しく動くか」を確認します。

### ✅ シナリオ1: 順当な勝利と決勝進出
- [ ] **手順**: 準決勝1 (SF1) で `Home: 2` - `Away: 0` と入力する。
- [ ] **確認**: 
    - [ ] 決勝戦の「ホーム」枠に SF1のホームチーム名が表示される。
    - [ ] 3位決定戦の「ホーム」枠に SF1のアウェイチーム名が表示される。

### ✅ シナリオ2: PK戦による決着
- [ ] **手順**: 準決勝2 (SF2) で `Home: 1` - `Away: 1` と入力し、[PK戦] にチェックを入れる。
- [ ] **手順**: PKスコアを `Home: 3` - `Away: 4` (アウェイ勝ち) と入力する。
- [ ] **確認**:
    - [ ] 決勝戦の「アウェイ」枠に SF2のアウェイチーム名が表示される。
    - [ ] 3位決定戦の「アウェイ」枠に SF2のホームチーム名が表示される。

### ✅ シナリオ3: 得点者と連動
- [ ] **手順**: 任意の試合で [+ 得点者を追加] をクリックする。
- [ ] **手順**: チームを選択し、選手名を入力（または選択）する。時間を入力する（例: "15"）。
- [ ] **確認**:
    - [ ] [得点ランキング] タブを開くと、その選手の名前と「1得点」が表示されている。
    - [ ] 同じ選手でもう1点追加すると、ランキングが「2得点」に更新される。

---

## 2. 異常系・意地悪テスト (Edge Cases)

「変な入力をしても、壊れないか」を確認します。

### ⚠️ テスト4: 無効な数値入力
- [ ] **スコア欄**: `-1` や `abc` を入力しようとする。
    - → 入力できない、または赤枠でエラー表示されること。
- [ ] **得点時間**: `前半` `後半` 以外のあり得ない文字を入れてみる。
    - → バリデーションが効いていること（必須ではないが、挙動確認）。

### ⚠️ テスト5: データの取り消し・矛盾
- [ ] **手順**: 一度入力した準決勝のスコアを全て消す（空欄にする）。
- [ ] **確認**: 決勝戦に進出していたチーム名が消え、「SF1勝者」などの初期表示に戻るか？（間違ったチームが残り続けないか）
- [ ] **手順**: 得点者を削除する（ゴミ箱アイコン）。
- [ ] **確認**: ランキングからその選手（または得点分）が即座に減るか？

---

## 3. 保存・出力テスト (Output Verification)

「作業内容が消えないか、正しく出力されるか」を確認します。

### 💾 テスト6: リロード対策（保存機能）
- [ ] **手順**: いくつか入力した状態で [保存] ボタンを押す。
- [ ] **手順**: ブラウザを更新（リロード）し、[読み込み] ボタンを押す。
- [ ] **確認**: 消えたはずのスコアや得点者が、完全に元通り復元されること。

### 📄 テスト7: 最終PDF用データ
- [ ] **手順**: [最終結果PDF用エクスポート] ボタンを押し、JSONをダウンロードする。
- [ ] **確認**: ダウンロードしたファイルをメモ帳などで開き、以下チェックする。
    - [ ] `ranking`: 1位〜4位のチーム名が正しいか。
    - [ ] `scorerRanking`: テストで入力した得点王がリストの先頭（または上位）にいるか。
    - [ ] `players`: 入力した優秀選手が含まれているか。
